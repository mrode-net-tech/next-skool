# Glossary

A single place to find every term used across the learning plan. Terms are listed alphabetically. Each entry has a short definition and a link to the day where it was first introduced.

> This glossary will grow as new phases are merged.

## A
- **App Router** — Modern routing in Next.js 13+ based on the `app/` directory and React Server Components. *(Day 81)*
- **async / await** — JavaScript syntax for working with Promises in a synchronous-looking way. *(Day 9)*
- **Auth.js** — Authentication library for Next.js (formerly NextAuth). *(Day 92)*

## B
- **bcrypt** — Library for securely hashing passwords. *(Day 21)*
- **BullMQ** — Redis-based job queue for Node.js. *(Day 111)*

## C
- **CI/CD** — Continuous Integration / Continuous Deployment. *(Day 106)*
- **Client Component** — A React component in Next.js that runs in the browser. Marked with `'use client'`. *(Day 83)*

## D
- **DDD** — Domain-Driven Design. *(Day 26)*
- **Docker** — Tool to package applications into containers. *(Day 16, Day 101)*
- **DTO** — Data Transfer Object. *(Day 67)*

## E
- **e2e (end-to-end) test** — A test that exercises the full system through its public interface. *(Day 76)*
- **Express** — Minimal HTTP framework for Node.js. *(Day 11)*

## J
- **JWT** — JSON Web Token. *(Day 22)*

## M
- **MSW** — Mock Service Worker. *(Day 54)*

## N
- **Next.js** — A React framework with built-in routing, SSR and server components. *(Day 81)*

## P
- **Pino** — Fast JSON logger for Node.js. *(Day 32)*
- **pnpm** — Fast, disk-efficient package manager. *(Day 61)*
- **Prisma** — Type-safe ORM for Node.js / TypeScript. *(Day 17)*

## R
- **React Testing Library (RTL)** — Library for testing React components from the user's perspective. *(Day 45)*
- **Repository (pattern)** — Abstraction over data access. *(Day 27)*

## S
- **Server Action** — Next.js feature: server-side function callable directly from a React component. *(Day 91)*
- **Server Component** — React component in Next.js that runs only on the server. *(Day 83)*
- **shadcn/ui** — Accessible styled React components built on Tailwind. *(Day 58)*
- **Supertest** — Library for testing HTTP servers in Node.js. *(Day 15)*

## T
- **Tailwind CSS** — A utility-first CSS framework. *(Day 56)*
- **TanStack Query** — Data-fetching library for React (formerly React Query). *(Day 51)*
- **Turborepo** — Build pipeline tool for monorepos. *(Day 64)*
- **TypeScript** — Typed superset of JavaScript. *(Day 4)*

## V
- **Vite** — Fast development server and bundler. *(Day 41)*
- **Vitest** — Modern test runner with a Jest-compatible API. *(Day 6)*

## Z
- **Zod** — TypeScript-first schema validation library. *(Day 14)*
